00071 class SortedCollection(object):
00072     '''Sequence sorted by a key function.
00073 
00074     SortedCollection() is much easier to work with than using bisect() directly.
00075     It supports key functions like those use in sorted(), min(), and max().
00076     The result of the key function call is saved so that keys can be searched
00077     efficiently.
00078 
00079     Instead of returning an insertion-point which can be hard to interpret, the
00080     five find-methods return a specific item in the sequence. They can scan for
00081     exact matches, the last item less-than-or-equal to a key, or the first item
00082     greater-than-or-equal to a key.
00083 
00084     Once found, an item's ordinal position can be located with the index() method.
00085     New items can be added with the insert() and insert_right() methods.
00086     Old items can be deleted with the remove() method.
00087 
00088     The usual sequence methods are provided to support indexing, slicing,
00089     length lookup, clearing, copying, forward and reverse iteration, contains
00090     checking, item counts, item removal, and a nice looking repr.
00091 
00092     Finding and indexing are O(log n) operations while iteration and insertion
00093     are O(n).  The initial sort is O(n log n).
00094 
00095     The key function is stored in the 'key' attibute for easy introspection or
00096     so that you can assign a new key function (triggering an automatic re-sort).
00097 
00098     In short, the class was designed to handle all of the common use cases for
00099     bisect but with a simpler API and support for key functions.

00102     >>> from operator import itemgetter

00107     ...         ('angela', 'jones', 28),

00110     ...     s.insert(record)
