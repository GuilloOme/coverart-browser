00079 
00080     SortedCollection() is much easier to work with than using bisect() directly.
00081     It supports key functions like those use in sorted(), min(), and max().
00082     The result of the key function call is saved so that keys can be searched
00083     efficiently.
00084 
00085     Instead of returning an insertion-point which can be hard to interpret, the
00086     five find-methods return a specific item in the sequence. They can scan for
00087     exact matches, the last item less-than-or-equal to a key, or the first item
00088     greater-than-or-equal to a key.
00089 
00090     Once found, an item's ordinal position can be located with the index() method.
00091     New items can be added with the insert() and insert_right() methods.
00092     Old items can be deleted with the remove() method.
00093 
00094     The usual sequence methods are provided to support indexing, slicing,
00095     length lookup, clearing, copying, forward and reverse iteration, contains
00096     checking, item counts, item removal, and a nice looking repr.
00097 
00098     Finding and indexing are O(log n) operations while iteration and insertion
00099     are O(n).  The initial sort is O(n log n).
00100 
00101     The key function is stored in the 'key' attibute for easy introspection or
00102     so that you can assign a new key function (triggering an automatic re-sort).
00103

00105     bisect but with a simpler API and support for key functions.

00116     ...     s.insert(record)
